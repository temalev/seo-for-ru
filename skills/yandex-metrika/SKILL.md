# Яндекс.Метрика — выгрузка статистики

Тянет данные из Яндекс.Метрики через [Stat API v1](https://yandex.ru/dev/metrika/ru/stat/). Чистый Node (≥18), без сторонних зависимостей.

## Когда использовать

Пользователь просит любые данные Метрики: трафик, визиты, источники, поисковые системы/запросы, популярные страницы, география, отказы, поведение.

## Предусловия

Конфиг берётся из `.env` в корне текущего проекта (или из env-переменных):

```
YANDEX_METRIKA_TOKEN=y0_...     # OAuth-токен, scope metrika:read
METRIKA_COUNTER_ID=12345678     # ID счётчика Метрики
```

`.env` должен быть в `.gitignore` — **токен никогда не коммитить и не хардкодить.**

**Где взять токен** (если нет): создать приложение на https://oauth.yandex.ru/ → тип «Для доступа к API или отладки» → право `metrika:read` → скопировать ClientID → открыть `https://oauth.yandex.ru/authorize?response_type=token&client_id=<ClientID>` → токен в URL после `#access_token=`.

**Где взять Counter ID:** в URL Метрики, либо в коде вставки на сайте — `ym(<ID>, "init", ...)`.

## Запуск

Скрипт `metrika.mjs` лежит рядом с этим SKILL.md.

**Если установлен как плагин** (`/plugin install`) — путь через `${CLAUDE_PLUGIN_ROOT}`:
```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/yandex-metrika/metrika.mjs" --report overview --days 30
```

**Если установлен как user-скилл** (симлинк в `~/.claude/skills/`) — `${CLAUDE_PLUGIN_ROOT}` не задан, запускай по пути скилла:
```bash
node ~/.claude/skills/yandex-metrika/metrika.mjs --report overview --days 30
```

Аргументы: `--report overview|sources|search|pages|geo|all` (дефолт `all`) · `--days N` (дефолт 30).

## Что отдаёт

| report | Данные |
|---|---|
| `overview` | визиты, посетители, просмотры, % отказов, глубина, ср. время |
| `sources` | источники трафика (поиск/прямые/соцсети/реклама) |
| `search` | поисковые системы + поисковые фразы (Яндекс часто прячет фразы — полнее в Вебмастере) |
| `pages` | топ страниц входа с % отказов |
| `geo` | топ городов |

## Важно: троттлинг Метрики

Stat API периодически отдаёт `400 "Query is too complicated"` даже на лёгких запросах — это **интермиттентный rate-троттлинг по частоте**, а не реальная сложность. Особенно чувствительны дименшн-запросы (`sources`/`search`/`pages`/`geo`).

В скрипте уже есть защита: интервал ≥1.1с между запросами + до 8 ретраев с растущим бэкоффом.

**Если упёрлось в троттл:** запускать секции по одной с паузами (`--report overview`, подождать, `--report sources`, …), либо дать API остыть 10–30 мин.
