# План: скилл yandex-webmaster

Второй инструмент коллекции seo-for-ru. Даёт то, что Метрика **не** показывает:
поисковые запросы с позициями/показами/кликами/CTR, ИКС, индексацию, диагностику.

## Зачем

- **Запросы с позициями** — по каким запросам показываемся, на какой позиции, CTR.
- **ИКС** — индекс качества сайта.
- **Индексация** — страниц в поиске.
- **Диагностика** — проблемы, найденные Яндексом.

SEO-применение: найти запросы на позициях 11–30 (дожать до топа) и запросы с
высокими показами но низким CTR (переписать title/description).

## Аутентификация

Вебмастер требует scope **`webmaster:read`** (у Метрики был только `metrika:read`).

- **A (рекомендую):** перевыпустить токен у того же OAuth-приложения с обоими правами
  `metrika:read` + `webmaster:read` → один `YANDEX_OAUTH_TOKEN` на оба скилла.
- **B:** отдельный `YANDEX_WEBMASTER_TOKEN`.

Скрипт читает `YANDEX_WEBMASTER_TOKEN` → fallback `YANDEX_OAUTH_TOKEN`.

## API (Webmaster v4 — `https://api.webmaster.yandex.net/v4/`)

| Эндпоинт | Зачем |
|---|---|
| `GET /user` | `user_id` (нужен для всех вызовов) |
| `GET /user/{uid}/hosts` | список сайтов → `host_id` |
| `.../hosts/{hid}/summary` | ИКС + кол-во проблем |
| `.../search-queries/popular` | запросы: показы/клики/позиция/CTR |
| `.../search-urls/in-search/history` | страниц в поиске (динамика) |
| `.../diagnostics` | проблемы сайта |

## Структура

```
skills/yandex-webmaster/
  ├── PLAN.md        # этот файл
  ├── SKILL.md       # автотриггер: «позиции», «запросы», «ИКС», «индексация»
  └── webmaster.mjs  # .env, throttle, ретраи, --report
```

Конфиг (.env проекта):
```
YANDEX_WEBMASTER_TOKEN=y0_...   # или YANDEX_OAUTH_TOKEN с webmaster:read
WEBMASTER_HOST=рк-тек.рф        # домен для выбора хоста (если их несколько)
```

Отчёты (`--report`): `queries` | `summary` | `indexing` | `diagnostics` | `all`.
