# PageSpeed Insights — скорость и Core Web Vitals

Закрывает техническое SEO, которого нет в Метрике и Вебмастере: **насколько быстро страница грузится и интерактивна**, и **что конкретно её тормозит**. Google PageSpeed Insights API запускает Lighthouse + берёт реальные поведенческие из Chrome UX Report.

## Когда использовать

- «Прогнать pagespeed по сайту»
- «Почему сайт медленный»
- «Что улучшить с точки зрения скорости»
- «Core Web Vitals», «LCP», «CLS», «INP», «TTI», «оптимизация страницы»
- «Технический аудит производительности»

## Конфиг

Ничего обязательного — API открытый, но **на практике стоит получить свой ключ**: общий no-key пул периодически выжигается другими пользователями интернета, и без ключа можно поймать `429 Quota exceeded` на ровном месте. Со своим ключом — 25 000 запросов/день только под вас.

```
PAGESPEED_API_KEY=AIza...    # из Google Cloud Console, поднимает rate-limit
```

### Как получить PAGESPEED_API_KEY (~5 минут, бесплатно)

1. **Зайти в [Google Cloud Console](https://console.cloud.google.com/)**. Если нет аккаунта Google Cloud — авторизуйтесь под своим Google-аккаунтом, согласитесь на ToS.

2. **Создать (или выбрать) проект.** Можно назвать «pagespeed» — содержимое не важно, нужна только оболочка для API-ключа.

3. **Включить PageSpeed Insights API.** Перейти в *APIs & Services → Library*, найти **«PageSpeed Insights API»**, нажать **Enable**. Без этого шага ключ создастся, но запросы будут возвращать `API has not been used in project ... before or it is disabled`.

4. **Создать API-ключ.** *APIs & Services → Credentials → + Create credentials → API key*.
   - **Name**: `PageSpeed Insights` (или любое — для себя).
   - **Select API restrictions**: выбрать **только** `PageSpeed Insights API` — ключ не сможет случайно дёргать другие сервисы Google, если засветится.
   - **Authenticate API calls through a service account**: НЕ ставить галку (это только для Vertex/Gemini).
   - **Application restrictions**: **None** (мы запускаем из терминала, IP плавающий).
   - **Create**.

5. **Сохранить ключ.** Cloud Console покажет его один раз — копируйте сразу. Положите в `.env` проекта, по которому собираетесь делать замеры:
   ```
   PAGESPEED_API_KEY=AIza...
   ```

6. (опц.) Если в Cloud у вас включён биллинг и хотите оставить ключ как есть — лимит и так бесплатный (25k/день, для замеров с горой). Если параноите — поставьте бюджетный алёрт на $1, никогда не сработает.

### Пограничные случаи

- **Без ключа**: можно пытаться, но при «Quota exceeded» — это не ваш баг, это общий пул. Решение: получить ключ.
- **Кириллические домены (IDN)**: скрипт автоматически конвертит в punycode перед отправкой. Lighthouse внутри Google не ест нативные IDN-URL.

## Запуск

```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/pagespeed/pagespeed.mjs" https://пример.рф/
node ~/.claude/skills/pagespeed/pagespeed.mjs https://пример.рф/ --strategy mobile
```

Аргументы:
- `<url>` — обязательный позиционный, полный URL включая `https://`
- `--strategy mobile|desktop|both` — дефолт `both`. На мобильном результаты обычно сильно хуже, и именно мобильный влияет на ранжирование Яндекса.

Один прогон стратегии — 5-15 секунд (Google запускает Lighthouse внутри). `both` ≈ 20-30с суммарно. Это нормально, не баг.

## Что отдаёт

Для каждой стратегии (мобильный/десктоп):

1. **Performance / SEO scores** — общие оценки Lighthouse (0-100, 🟢 ≥ 90, 🟡 50-89, 🔴 < 50).
2. **Core Web Vitals (лаб-метрики Lighthouse)** — LCP, CLS, TBT, FCP, SI, TTI с нормами.
3. **Реальные пользователи (Chrome UX Report)** — LCP/CLS/INP/FCP по реальным посетителям за последние 28 дней. Если у сайта мало трафика, CrUX данных не будет (это норма).
4. **🎯 Что улучшить** — топ-5 рекомендаций Lighthouse по убыванию импакта (с оценкой экономии в миллисекундах).
5. **⚠️ SEO-замечания** — Lighthouse-проверки SEO, которые провалились (viewport, canonical, alt-теги, размер шрифта и т.п.).

## Лаб vs реальные пользователи

Lighthouse-метрики — это **синтетический** замер на эталонной мобилке Moto G Power с дросселированным каналом. Хорошо для диагностики и сравнения изменений.

Chrome UX Report — **реальные** анонимные данные с Chrome-пользователей за 28 дней. Это то, что **видит Google Search** при ранжировании. Если CrUX и Lighthouse сильно расходятся — верить CrUX.

## Что важно для русского SEO

Яндекс тоже использует скорость и удобство в ранжировании, поэтому Core Web Vitals = универсальный SEO-сигнал (не только Google). Особенно мобильный LCP — мобильный поиск рунета это 60-70%.
