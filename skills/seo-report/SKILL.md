# SEO кросс-отчёт — позиции × спрос × поведение

Сводит **одну обязательную и две опциональные оси** по сайту в один отчёт. Чистый Node (≥18), без зависимостей.

| Ось | Что даёт | Источник | Обязат. |
|---|---|---|---|
| Позиции | показы, клики, CTR, средняя позиция по запросам | Вебмастер | да |
| Спрос | частотность фразы в месяц (totalCount) | Wordstat | нет |
| Поведение | визиты + отказы по поисковой фразе | Метрика (Stat API) | нет |

Каждая опциональная ось — **best-effort**: нет токена/доступа/данных → колонки этой оси `·`, бакет этой оси скрывается, остальное работает.

## Когда использовать

Когда нужно **приоритизировать**, а не просто посмотреть позиции:
- «какие запросы 11–30 дожимать в первую очередь» — по спросу/визитам/показам
- запросы с низким CTR для переписывания сниппета
- высокоспросные запросы, где сайт ранжируется слабо → новый контент/страница
- запросы, по которым трафик идёт, но отказы высокие → чинить посадочную

Если нужна только одна сторона — бери [yandex-webmaster](../yandex-webmaster/SKILL.md) или [yandex-metrika](../yandex-metrika/SKILL.md).

## Конфиг (.env проекта-сайта)

```
# Вебмастер (обязательно)
YANDEX_WEBMASTER_TOKEN=y0_...        # или YANDEX_OAUTH_TOKEN со scope webmaster:read
WEBMASTER_HOST=пример.рф             # если сайтов несколько

# Метрика (опционально — добавляет ось поведения)
YANDEX_METRIKA_TOKEN=y0_...          # или YANDEX_OAUTH_TOKEN со scope metrika:read
METRIKA_COUNTER_ID=12345678

# Wordstat (опционально — добавляет ось спроса). Один из двух наборов:
YANDEX_WORDSTAT_TOKEN=y0_...                       # A: прямой Wordstat API (OAuth)
# или
YANDEX_CLOUD_API_KEY=AQVN... YANDEX_CLOUD_FOLDER_ID=b1g...   # B: Cloud Search API
WORDSTAT_REGIONS=11                  # id регионов (11=Рязань, 213=Москва); пусто = вся Россия
```

Удобно перевыпустить **один** `YANDEX_OAUTH_TOKEN` со scope `metrika:read` + `webmaster:read` — подхватится как fallback для обеих осей.

### Доступ к Wordstat

Wordstat доступен двумя дверями, скрипт выберет первый доступный набор кредов:

- **A — прямой Wordstat API** (`api.wordstat.yandex.net`): OAuth `Bearer`-токен. Заполнить форму разблокировки внизу [wordstat.yandex.ru](https://wordstat.yandex.ru) (~день на аппрув), потом OAuth-флоу. Исторически бесплатный, лимит 10/с, 1000/день. С появлением платности Cloud-варианта **скорее всего тоже тарифицируется** — проверьте у Яндекса актуальный статус перед использованием.
- **B — Yandex Cloud Search API** (`searchapi.api.cloud.yandex.net/v2`): API-ключ AI Studio (`Api-Key`) + folderId. **Платный с 18 мая 2026** — см. [тарификация](https://aistudio.yandex.ru/docs/ru/search-api/pricing.html). Текущая цена `GetTop`: **20 ₽ / 1000 запросов** (с НДС). Один прогон cross.mjs ≈ 65 запросов ≈ 1.3 ₽. Требует на облаке: (1) роль `search-api.webSearch.user` сервисному аккаунту ключа, (2) **активный биллинг-аккаунт** (без него 403 PERMISSION_DENIED даже с правильной ролью), (3) у новых пользователей Cloud обычно есть тестовый грант ~4000 ₽ на 60 дней — хватает на тысячи прогонов.

## Запуск

```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/seo-report/cross.mjs"
node ~/.claude/skills/seo-report/cross.mjs --days 14 --order demand
```

Аргументы:
- `--days N` : окно (дефолт 7; конец сдвигается на -2 дня — последние дни Вебмастер ещё не обработал)
- `--limit N` : строк в основной таблице (дефолт 50)
- `--order` : `shows` | `clicks` | `position` | `ctr` | `demand` | `visits` (дефолт `shows`)
- `--no-filter` : выключить спам-фильтр (см. ниже)
- `--diff` : показать дельту с предыдущим снимком (см. «Снимки и diff»)
- `--no-save` : не сохранять снимок этого прогона

## Что отдаёт

1. **Таблица** запрос → показы, клики, CTR, позиция, спрос, визиты Метрики, % отказов.
2. **Действия** — до четырёх плюс один бакетов:
   - 🎯 **Дожать в топ** (поз 11–30) — приоритет: спрос → визиты → показы.
   - ✏️ **Переписать сниппет** — ранжируется (поз ≤ 15), CTR < 3%, показов ≥ 50.
   - 💎 **Высокий спрос — слабая видимость** — спрос есть, позиция > 20 (нужен Wordstat).
   - 🚧 **Слабая посадочная** — визиты ≥ 5, отказы ≥ 50% (нужна Метрика).
   - 🚫 **Похоже на накрутку** — что отфильтровано спам-эвристикой (см. ниже).

## Спам-фильтр запросов

Конкуренты/боты часто гонят показы по мусорным фразам, чтобы засветиться в чужой выдаче. Это **засирает бакет «дожать в топ»** настолько, что реальные точки роста теряются. Скрипт по умолчанию прячет такие фразы из таблицы и бакетов, но не молча — сводит их в 🚫-бакет с причиной.

Три эвристики (все мягкие, без ML):

1. **`top-no-clicks`** — позиция ≤ 12, показов ≥ 3, кликов **0**. На первой странице запрос с заметными показами должен давать хотя бы 1 клик; ноль — почти всегда аномалия.
2. **`glued`** — слитная кириллическая опечатка ≥ 14 букв без пробелов + 0 кликов (`проституткиирязань`, `теклаюрф`-подобные).
3. **`blocklist`** — пользовательский regex `SEO_BLOCKLIST` в `.env`. Под сайт-специфичную накрутку (домены конкурентов, телефоны, бренды).

Пример для сайта вроде рк-тек.рф, по которому гонят конкурента `skupik.ru`:

```
SEO_BLOCKLIST=skupik|74994440|проститут|казино|порн
```

`--no-filter` показывает накрутку в основной таблице наравне с остальным. Пороги — константы в начале `cross.mjs`.

## Снимки и diff (трекинг во времени)

Каждый прогон тихо сохраняет JSON-снимок в `.seo-snapshots/<домен>/<timestamp>.json` (директорию можно сменить через `SEO_SNAPSHOT_DIR`). В снимке: все запросы Вебмастера (включая отфильтрованную накрутку), ИКС, полная диагностика, метаданные. Это git-friendly trail: гоните раз в неделю — через месяц у вас есть **история** позиций без какой-либо БД и подписок.

С флагом `--diff` отчёт дополнительно сравнивает текущее состояние с **последним предыдущим снимком** и печатает что изменилось:

- 📈 **Поднялись в выдаче** — позиция улучшилась на ≥ 1.
- 📉 **Упали в выдаче** — позиция ухудшилась.
- 🆕 **Новые в индексе** — фразы, которых не было в прошлом снимке.
- 👋 **Выпали из индекса** — фразы, исчезнувшие из Вебмастера.
- ✅ **Решённые проблемы** / 🚨 **Новые проблемы** — переходы в диагностике (PRESENT ↔ ABSENT/UNDEFINED).
- 📊 **ИКС** — дельта индекса качества.

`--no-save` отключает сохранение для разовых/тестовых прогонов. Папку `.seo-snapshots/` имеет смысл добавить в `.gitignore`, если не хотите коммитить историю прогонов в репо проекта (но JSON компактные, обычно ≤ 5 KB на снимок, так что коммитить тоже нормальный вариант).

## Заметка по троттлингу Метрики

Stat API на тяжёлых счётчиках устойчиво отбивает запросы с измерением `searchPhrase` ошибкой 400 «Query is too complicated» — это **детерминированный** отказ для данной формы запроса, ретрай его не пробивает (каждый отказ ещё и висит ~11с). Поэтому ось поведения тут **fail-fast**: один запрос, на ошибке деградируем; ретрай только настоящий 429 по `Retry-After`. На лёгких счётчиках всё работает; Яндекс часто прячет фразы, так что матч с Вебмастером бывает частичным или пустым — это нормально.

Пороги бакетов — простые константы в начале `cross.mjs`; правьте под свой сайт.
