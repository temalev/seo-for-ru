# SEO кросс-отчёт — позиции × поведение

Джойнит запросы из [Вебмастера](../yandex-webmaster/SKILL.md) (позиции/показы/клики/CTR) с поведением из [Метрики](../yandex-metrika/SKILL.md) (визиты/отказы) по тексту запроса. Чистый Node (≥18), без зависимостей.

## Когда использовать

Когда нужно **связать** две стороны, которые по отдельности не стыкуются:
- «по каким запросам показываемся **и** что делают пришедшие»
- запросы для дожима в топ, для переписывания сниппета, слабые посадочные
- единый SEO-дашборд вместо двух разрозненных отчётов

Если нужна только одна сторона — бери [yandex-webmaster](../yandex-webmaster/SKILL.md) (позиции) или [yandex-metrika](../yandex-metrika/SKILL.md) (трафик).

## Предусловия

Нужны конфиги **обоих** скиллов в `.env` корня проекта:

```
# Вебмастер (scope webmaster:read)
YANDEX_WEBMASTER_TOKEN=y0_...      # или YANDEX_OAUTH_TOKEN с обоими scope
WEBMASTER_HOST=рк-тек.рф           # если сайтов несколько

# Метрика (scope metrika:read)
YANDEX_METRIKA_TOKEN=y0_...        # или тот же YANDEX_OAUTH_TOKEN
METRIKA_COUNTER_ID=12345678
```

Удобнее всего перевыпустить **один** `YANDEX_OAUTH_TOKEN` с обоими правами `metrika:read` + `webmaster:read` — он подхватится как fallback для обеих сторон.

## Запуск

```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/seo-report/cross.mjs"
node ~/.claude/skills/seo-report/cross.mjs --days 14 --order position
```

Аргументы:
- `--days N` : окно (дефолт 7; конец сдвигается на -2 дня — последние дни Вебмастер ещё не обработал). Без `--days` берётся свежее окно Вебмастера.
- `--limit N` : сколько строк в основной таблице (дефолт 50)
- `--order` : `shows` | `clicks` | `position` | `ctr` (дефолт `shows`)

## Что отдаёт

1. **Таблица** запрос → показы, клики, CTR, средняя позиция + визиты и % отказов из Метрики (где сошлось).
2. **Действия** — три бакета:
   - 🎯 **Дожать в топ** — позиция 11–30 с заметными показами.
   - ✏️ **Переписать сниппет** — ранжируется (поз ≤ 15), но CTR низкий при достаточных показах.
   - 🚧 **Слабая посадочная** — трафик из Метрики есть, но отказы высокие (только если Метрика сматчила запросы).

## Важно про матч с Метрикой

Яндекс **массово скрывает** поисковые фразы в Метрике, поэтому матч по тексту запроса бывает частичным или пустым. Это нормально: отчёт устроен так, что выводы по позициям/CTR (Вебмастер) работают **всегда**, а поведение (Метрика) накладывается как обогащение там, где запрос сошёлся. В шапке таблицы печатается «матч N из M».

Пороги бакетов — простые эвристики, заданы константами в начале `cross.mjs`; правьте под свой сайт.
