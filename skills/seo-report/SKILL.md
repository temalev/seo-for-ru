# SEO кросс-отчёт — позиции × спрос

Джойнит запросы из [Вебмастера](../yandex-webmaster/SKILL.md) (позиции/показы/клики/CTR) со спросом из Wordstat (частотность запроса в месяц) по тексту запроса. Чистый Node (≥18), без зависимостей.

## Когда использовать

Когда нужно **приоритизировать**, а не просто посмотреть позиции:
- «какие запросы 11–30 дожимать в первую очередь» → те, где выше спрос
- запросы с низким CTR для переписывания сниппета (по убыванию спроса)
- высокоспросные запросы, где сайт ранжируется слабо → новый контент/страница

Если нужна только одна сторона — бери [yandex-webmaster](../yandex-webmaster/SKILL.md) (позиции) или [yandex-metrika](../yandex-metrika/SKILL.md) (трафик/поведение).

## Почему спрос из Wordstat, а не поведение из Метрики

Stat API Метрики массово **прячет** поисковые фразы Яндекса и жёстко **троттлит** запросы с измерением `searchPhrase` (ошибка «Query is too complicated»). Серьёзные RU-SEO-инструменты фразы из Метрики не тянут вовсе: авторитетный источник запросов и позиций — Вебмастер, а спрос берут из Wordstat. Поэтому здесь поведенческая ось заменена на спрос.

## Предусловия

Обязателен токен **Вебмастера**. Wordstat — опционально (без него отчёт деградирует к чистому Вебмастеру). Доступ к Wordstat — **на выбор**, два равноценных по данным варианта; скрипт берёт первый доступный в `.env`:

```
# Вебмастер (обязательно)
YANDEX_WEBMASTER_TOKEN=y0_...      # или YANDEX_OAUTH_TOKEN со scope webmaster:read
WEBMASTER_HOST=рк-тек.рф           # если сайтов несколько

# Wordstat — вариант A: прямой Wordstat API (OAuth)
YANDEX_WORDSTAT_TOKEN=y0_...       # Bearer-токен Wordstat API

# Wordstat — вариант B: Yandex Cloud Search API (ключ AI Studio + folder)
YANDEX_CLOUD_API_KEY=AQVN...       # или YC_API_KEY
YANDEX_CLOUD_FOLDER_ID=b1g...      # или YC_FOLDER_ID

# Общее для Wordstat
WORDSTAT_REGIONS=11                # id регионов через запятую (11=Рязань, 213=Москва); пусто = вся Россия
WORDSTAT_DEVICES=all               # all|desktop|phone|tablet (дефолт all; только для A)
```

### Вариант A — прямой Wordstat API (`api.wordstat.yandex.net`)

Отдельный OAuth-токен (метрика/вебмастер-токены не подойдут):

1. **Запросить доступ** — на [wordstat.yandex.ru](https://wordstat.yandex.ru) внизу страницы заполнить форму на разблокировку API (аппрув обычно в течение дня).
2. **Сгенерировать OAuth-токен** — `client_id` на [oauth.yandex.ru](https://oauth.yandex.ru), пройти `https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID`, забрать `access_token` → в `.env` как `YANDEX_WORDSTAT_TOKEN` (живёт 1 год). Лимит 10/с, 1000/день.

### Вариант B — Yandex Cloud Search API (`searchapi.api.cloud.yandex.net/v2`)

Тот же Wordstat через [Search API](https://aistudio.yandex.ru/docs/ru/search-api/concepts/wordstat) (Preview, **платный** по запросам). Удобнее, если уже есть аккаунт Yandex Cloud / AI Studio — ключ выдаётся сразу, без ожидания аппрува:

1. Получить **API-ключ AI Studio** (имеет нужные роли) → `YANDEX_CLOUD_API_KEY`.
2. Узнать **id каталога** (folder) Cloud → `YANDEX_CLOUD_FOLDER_ID`.

Авторизация `Authorization: Api-Key <ключ>`; `folderId` уходит в теле запроса. Роль: `search-api.webSearch.user`.

## Запуск

```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/seo-report/cross.mjs"
node ~/.claude/skills/seo-report/cross.mjs --days 14 --order demand
```

Аргументы:
- `--days N` : окно Вебмастера (дефолт 7; конец сдвигается на -2 дня — последние дни Вебмастер ещё не обработал). Без `--days` берётся свежее окно.
- `--limit N` : сколько строк в основной таблице (дефолт 50)
- `--order` : `shows` | `clicks` | `position` | `ctr` | `demand` (дефолт `shows`)

## Что отдаёт

1. **Таблица** запрос → показы, клики, CTR, средняя позиция + спрос/мес из Wordstat (где сошлось).
2. **Действия** — три бакета:
   - 🎯 **Дожать в топ** — позиция 11–30, отсортированы по спросу (или показам без Wordstat): максимум недобранного трафика сверху.
   - ✏️ **Переписать сниппет** — ранжируется (поз ≤ 15), но CTR низкий при достаточных показах.
   - 💎 **Высокий спрос — слабая видимость** — заметный спрос, но позиция > 20 (только при данных Wordstat): кандидаты на новый контент/страницу.

## Важно про спрос

Wordstat запрашивается по одному запросу на фразу (поле `totalCount` — широкая частотность), только для показанных строк и кандидатов на дожим, с потолком обращений за прогон. Спрос — обогащение: выводы по позициям/CTR (Вебмастер) работают **всегда**, спрос накладывается там, где Wordstat доступен. Регион берётся из `WORDSTAT_REGIONS`; для локального бизнеса задайте свой регион — спрос «вся Россия» сильно отличается от регионального.

Пороги бакетов — простые эвристики, заданы константами в начале `cross.mjs`; правьте под свой сайт.
