# SEO кросс-отчёт — позиции × спрос × поведение

Сводит **одну обязательную и две опциональные оси** по сайту в один отчёт. Чистый Node (≥18), без зависимостей.

| Ось | Что даёт | Источник | Обязат. |
|---|---|---|---|
| Позиции | показы, клики, CTR, средняя позиция по запросам | Вебмастер | да |
| Спрос | частотность фразы в месяц (totalCount) | Wordstat | нет |
| Поведение | визиты + отказы по поисковой фразе | Метрика (Stat API) | нет |

Каждая опциональная ось — **best-effort**: нет токена/доступа/данных → колонки этой оси `·`, бакет этой оси скрывается, остальное работает.

## Когда использовать

Когда нужно **приоритизировать**, а не просто посмотреть позиции:
- «какие запросы 11–30 дожимать в первую очередь» — по спросу/визитам/показам
- запросы с низким CTR для переписывания сниппета
- высокоспросные запросы, где сайт ранжируется слабо → новый контент/страница
- запросы, по которым трафик идёт, но отказы высокие → чинить посадочную

Если нужна только одна сторона — бери [yandex-webmaster](../yandex-webmaster/SKILL.md) или [yandex-metrika](../yandex-metrika/SKILL.md).

## Конфиг (.env проекта-сайта)

```
# Вебмастер (обязательно)
YANDEX_WEBMASTER_TOKEN=y0_...        # или YANDEX_OAUTH_TOKEN со scope webmaster:read
WEBMASTER_HOST=пример.рф             # если сайтов несколько

# Метрика (опционально — добавляет ось поведения)
YANDEX_METRIKA_TOKEN=y0_...          # или YANDEX_OAUTH_TOKEN со scope metrika:read
METRIKA_COUNTER_ID=12345678

# Wordstat (опционально — добавляет ось спроса). Один из двух наборов:
YANDEX_WORDSTAT_TOKEN=y0_...                       # A: прямой Wordstat API (OAuth)
# или
YANDEX_CLOUD_API_KEY=AQVN... YANDEX_CLOUD_FOLDER_ID=b1g...   # B: Cloud Search API
WORDSTAT_REGIONS=11                  # id регионов (11=Рязань, 213=Москва); пусто = вся Россия
```

Удобно перевыпустить **один** `YANDEX_OAUTH_TOKEN` со scope `metrika:read` + `webmaster:read` — подхватится как fallback для обеих осей.

### Доступ к Wordstat

Wordstat доступен двумя дверями, скрипт выберет первый доступный набор кредов:

- **A — прямой Wordstat API** (`api.wordstat.yandex.net`): OAuth `Bearer`-токен. Заполнить форму разблокировки внизу [wordstat.yandex.ru](https://wordstat.yandex.ru) (~день на аппрув), потом OAuth-флоу. Лимит 10/с, 1000/день.
- **B — Yandex Cloud Search API** (`searchapi.api.cloud.yandex.net/v2`, Preview, платный): API-ключ AI Studio + folderId. Удобнее, если уже есть аккаунт Cloud. Требует роль `search-api.webSearch.user` сервисному аккаунту ключа и активный биллинг.

## Запуск

```bash
node "${CLAUDE_PLUGIN_ROOT}/skills/seo-report/cross.mjs"
node ~/.claude/skills/seo-report/cross.mjs --days 14 --order demand
```

Аргументы:
- `--days N` : окно (дефолт 7; конец сдвигается на -2 дня — последние дни Вебмастер ещё не обработал)
- `--limit N` : строк в основной таблице (дефолт 50)
- `--order` : `shows` | `clicks` | `position` | `ctr` | `demand` | `visits` (дефолт `shows`)

## Что отдаёт

1. **Таблица** запрос → показы, клики, CTR, позиция, спрос, визиты Метрики, % отказов.
2. **Действия** — до четырёх бакетов в зависимости от доступных данных:
   - 🎯 **Дожать в топ** (поз 11–30) — приоритет: спрос → визиты → показы.
   - ✏️ **Переписать сниппет** — ранжируется (поз ≤ 15), CTR < 3%, показов ≥ 50.
   - 💎 **Высокий спрос — слабая видимость** — спрос есть, позиция > 20 (нужен Wordstat).
   - 🚧 **Слабая посадочная** — визиты ≥ 5, отказы ≥ 50% (нужна Метрика).

## Заметка по троттлингу Метрики

Stat API на тяжёлых счётчиках устойчиво отбивает запросы с измерением `searchPhrase` ошибкой 400 «Query is too complicated» — это **детерминированный** отказ для данной формы запроса, ретрай его не пробивает (каждый отказ ещё и висит ~11с). Поэтому ось поведения тут **fail-fast**: один запрос, на ошибке деградируем; ретрай только настоящий 429 по `Retry-After`. На лёгких счётчиках всё работает; Яндекс часто прячет фразы, так что матч с Вебмастером бывает частичным или пустым — это нормально.

Пороги бакетов — простые константы в начале `cross.mjs`; правьте под свой сайт.
